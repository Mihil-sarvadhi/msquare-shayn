What changes under the hood

7 Days → ?range=7d
|
30 Days → ?range=30d
|
Custom → ?startDate=2024-04-01&endDate=2024-04-20

The backend needs to handle the two new query params (startDate + endDate) on all dashboard and analytics endpoints. The range param is omitted when custom dates are set.

One global range vs. per-page range

G

Global — one filter rules all pages

Single Redux state. Switching from Dashboard → Marketing keeps your date selection. Changing range on any page changes it everywhere. Simpler, more coherent experience.

P

Per-page — each page remembers its own range

Dashboard and Analytics slices stay separate. You can compare "7 Days" on Dashboard vs "30 Days" on Marketing. More flexible, but adds complexity and can confuse users about which range is active.

My recommendation: Global. A D2C manager usually wants to compare metrics across pages for the same time window.
